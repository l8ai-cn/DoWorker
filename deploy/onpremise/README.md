# Agent Cloud OnPremise Deployment

Private deployment solution for intranet environments with an application DNS
name and a server IP for Runner connectivity.

## System Requirements

- Docker 20.10+
- Docker Compose V2
- 4 GB+ RAM
- 20 GB+ disk space
- Application DNS, for example `app.agentcloud.internal`
- Wildcard preview DNS, for example `*.preview.agentcloud.internal`

## Quick Start

### Option 1: Installation Script (Recommended)

```bash
# Extract the deployment package
tar -xzf agentcloud-onpremise-*.tar.gz
cd agentcloud-onpremise

# One-command install
./scripts/install.sh --ip 192.168.1.100 --host app.agentcloud.internal --preview-origin http://preview.agentcloud.internal

# Custom ports
./scripts/install.sh --ip 192.168.1.100 --host app.agentcloud.internal --preview-origin http://preview.agentcloud.internal:8080 --http-port 8080 --grpc-port 9443
```

### Option 2: Manual Installation

```bash
# 1. Load Docker images
./scripts/load-images.sh

# 2. Generate SSL certificates
./scripts/generate-certs.sh 192.168.1.100

# 3. Create configuration file
cp .env.template .env
# Edit .env and fill in the configuration

# 4. Start services
docker compose up -d

# 5. Run database migrations
docker compose exec backend migrate -path /app/migrations \
    -database "postgres://agentcloud:<DB_PASSWORD>@postgres:5432/agentcloud?sslmode=disable" up

# 6. Import seed data
docker compose exec -T postgres psql -U agentcloud -d agentcloud < seed/onpremise-seed.sql
```

## Runner Registration

On the machine where you want to run AI Agent tasks:

```bash
# Install Runner (macOS / Linux)
curl -fsSL https://agentcloud.ai/install.sh | sh

# Register Runner
agentcloud-runner register --server http://192.168.1.100 --token onpremise-runner-token

# Start Runner
agentcloud-runner run
```

## Common Commands

```bash
# View service status
docker compose ps

# View logs
docker compose logs -f              # All services
docker compose logs -f backend      # Backend only
docker compose logs -f web          # Web only

# Restart a service
docker compose restart backend

# Stop services
docker compose down

# Stop and remove all data
docker compose down -v

# Restart after config changes
docker compose up -d
```

## Data Backup

### Backup Database

```bash
# Backup
docker compose exec postgres pg_dump -U agentcloud agentcloud > backup.sql

# Restore
docker compose exec -T postgres psql -U agentcloud -d agentcloud < backup.sql
```

### Backup All Data

```bash
# Stop services
docker compose down

# Backup volumes
docker run --rm -v agentcloud_postgres_data:/data -v $(pwd):/backup alpine \
    tar czf /backup/postgres_backup.tar.gz -C /data .

docker run --rm -v agentcloud_minio_data:/data -v $(pwd):/backup alpine \
    tar czf /backup/minio_backup.tar.gz -C /data .

# Start services
docker compose up -d
```

## Troubleshooting

### Services Won't Start

```bash
# Check Docker status
docker info

# View detailed logs
docker compose logs --tail=100

# Check port conflicts
netstat -tlnp | grep -E ':(80|9443|5432|6379|9000)\s'
```

### Database Connection Failed

```bash
# Check PostgreSQL status
docker compose exec postgres pg_isready -U agentcloud

# View database logs
docker compose logs postgres
```

### Runner Can't Connect

1. Confirm firewall allows ports: 80 (HTTP) and 9443 (gRPC)
2. Verify the SSL certificate includes the server IP
3. Check Backend logs: `docker compose logs backend | grep -i grpc`

## Security Recommendations

1. **Change default password**: Change the admin password immediately after first login
2. **Firewall configuration**: Only open necessary ports (80, 9443)
3. **Regular backups**: Regularly back up the database and object storage
4. **Certificate renewal**: SSL certificates are valid for 1 year — renew before expiry
5. **Audit logs**: Review Admin audit logs regularly

## Version Upgrade

```bash
# 1. Backup data
./scripts/backup.sh  # (if available)

# 2. Load new version images
./scripts/load-images.sh

# 3. Recreate containers
docker compose up -d

# 4. Run migrations (if needed)
docker compose exec backend migrate -path /app/migrations \
    -database "postgres://agentcloud:${DB_PASSWORD}@postgres:5432/agentcloud?sslmode=disable" up
```
